# MudBlazor Theming & Customization Mastery  

## 1. Core Theming Philosophy (Always Follow)
- **MudThemeProvider is mandatory** â€” there must be exactly **one** instance in the entire app (usually in `MainLayout.razor` or `App.razor`).
- Never hard-code colors, fonts, shadows, or spacing outside of the `MudTheme` object.
- All MudBlazor components automatically respect the active `MudTheme` via CSS variables (`--mud-palette-*`, `--mud-typography-*`, etc.).
- Support **Light + Dark** mode natively (and system preference).
- Use `MudTheme` class for everything â€” it is the single source of truth.
- Prefer `MudColor` for any color definitions (supports RGBA, HEX, HSL, lighten/darken).

## 2. Required Providers Setup (2026 Standard)

```razor
<!-- MainLayout.razor or App.razor -->
<MudThemeProvider @bind-Theme="@CurrentTheme"
                  @bind-CurrentPalette="@CurrentPalette"
                  ObserveSystemDarkModeChange="true"
                  IsDarkMode="@IsDarkMode">
    <MudPopoverProvider />
    <MudDialogProvider />
    <MudSnackbarProvider />
    <MudLayout>
        <!-- App content -->
    </MudLayout>
</MudThemeProvider>
```

**Key new 2026 parameters on MudThemeProvider:**
- `@bind-CurrentPalette` â€” two-way binding for live palette changes
- `ObserveSystemDarkModeChange="true"` â€” automatically follows OS dark mode
- `IsDarkMode` â€” explicit control with persistence

## 3. Creating & Applying a Custom Theme

```razor
@code {
    private MudTheme CurrentTheme = new MudTheme()
    {
        PaletteLight = new PaletteLight()
        {
            Primary = "#4CAF50",           // or MudColor.FromString("#4CAF50")
            Secondary = "#FF9800",
            Background = "#F5F5F5",
            AppBarBackground = "#FFFFFF",
            DrawerBackground = "#FFFFFF",
            // ... full override possible
        },
        PaletteDark = new PaletteDark()
        {
            Primary = "#66BB6A",
            Secondary = "#FFB74D",
            Background = "#121212",
            Surface = "#1E1E1E",
            // ... 
        },

        Typography = new Typography()
        {
            Default = new TypographyDefault()
            {
                FontFamily = new[] { "Roboto", "Helvetica", "Arial", "sans-serif" },
                FontSize = "1rem",
                FontWeight = 400
            },
            H1 = new TypographyH1() { FontSize = "3rem", FontWeight = 500 },
            // override any heading, body, button, caption, etc.
        },

        LayoutProperties = new LayoutProperties()
        {
            DefaultBorderRadius = "4px",
            DrawerWidthLeft = "280px",
            DrawerWidthRight = "280px",
            AppbarHeight = "64px"
        },

        Shadows = new Shadow()
        {
            // override elevation shadows if needed
        },

        ZIndex = new ZIndex() { Drawer = 1100, AppBar = 1200, /* etc. */ }
    };

    private bool IsDarkMode = false;
    private Palette CurrentPalette => IsDarkMode ? CurrentTheme.PaletteDark! : CurrentTheme.PaletteLight!;

    private void ToggleDarkMode()
    {
        IsDarkMode = !IsDarkMode;
        StateHasChanged();
    }
}
```

## 4. Full Theming Properties You Can Customize (Official Structure)

### 4.1 Palette (Most Important)
- **PaletteLight** + **PaletteDark** (both inherit from `Palette`)
- 30+ color slots: `Primary`, `Secondary`, `Tertiary`, `Background`, `Surface`, `DrawerBackground`, `AppBarBackground`, `ActionDefault`, `Success`, `Warning`, `Error`, `Info`, `TextPrimary`, `TextSecondary`, `Overlay`, `OverlayDark`, `OverlayLight`, etc.
- Every color has automatic **Lighten/Darken** variants generated by MudBlazor.
- All exposed as CSS variables: `--mud-palette-primary`, `--mud-palette-primary-darken-1`, etc.

### 4.2 Typography
- `Default`, `H1`â€“`H6`, `Subtitle1`â€“`Subtitle2`, `Body1`â€“`Body2`, `Button`, `Caption`, `Overline`
- Properties: `FontFamily`, `FontSize`, `FontWeight`, `LineHeight`, `LetterSpacing`, `TextTransform`

### 4.3 LayoutProperties
- `DefaultBorderRadius`, `DrawerWidthLeft/Right`, `AppbarHeight`, `AppbarMinHeight`, etc.

### 4.4 Shadows
- Full control over all 25 elevation levels (`Elevation0` to `Elevation24`)

### 4.5 Z-Index
- Fine-tune stacking order for `Drawer`, `AppBar`, `Popover`, `Dialog`, `Snackbar`, etc.

### 4.6 Pseudo CSS & Scopes (Advanced)
- Use `PseudoCss` and `PseudoCssScope` for component-specific overrides without breaking the global theme.

## 5. Dark / Light Mode Mastery (2026 Best Practices)

```razor
<!-- Toggle button example -->
<MudSwitch @bind-Value="@IsDarkMode" Color="Color.Primary">
    @if (IsDarkMode) { <MudText>Dark Mode</MudText> } else { <MudText>Light Mode</MudText> }
</MudSwitch>
```

**Advanced options:**
- `ObserveSystemDarkModeChange="true"` on `MudThemeProvider`
- Persist user choice with `LocalStorage` or `ProtectedLocalStorage`
- Bind `@bind-Theme` for completely separate light/dark themes
- Use `CurrentPalette` binding for live preview tools

## 7. Advanced Techniques & Best Practices

1. **CSS Variable Overrides** (last resort)
   ```css
   :root {
       --mud-palette-primary: #4CAF50;
   }
   ```

2. **Component-level theming**
   - Most components accept `Color`, `Variant`, `Class`, `Style`
   - Use `mud-theme-primary`, `mud-theme-surface`, etc. utility classes

3. **Custom component wrappers**
   ```razor
   <!-- MyCustomButton.razor -->
   <MudButton Color="Color.Primary" Variant="Variant.Filled" ...>
   ```

4. **Performance tip:** Avoid recreating `MudTheme` object on every render â€” keep it as a static or singleton field.

5. **Never** mix multiple `MudThemeProvider` instances.

6. **Validation:** Always test both light and dark mode + mobile responsiveness.

## 8. Common Theming Patterns You Must Master

- Full dark/light toggle with persistence
- Brand-specific palette (company colors)
- Custom typography (corporate fonts)
- Responsive drawer/appbar heights
- High-contrast accessibility themes
- Multi-brand theming (switch themes at runtime)
- Export/import themes via JSON (using ThemeManager)

## 9. Forbidden Practices in Theming
- Hard-coding `#hex` colors anywhere in Razor or CSS files
- Using external CSS frameworks for colors/spacing
- Forgetting to update both `PaletteLight` AND `PaletteDark`
- Creating more than one `MudThemeProvider`
- Ignoring CSS custom properties (`--mud-*`)
